## 📅 每周操作流程

### 周日晚上（收盘后）

#### 1. 更新数据
```bash
# 确保bitcoin_weekly.csv已更新到本周
python ms_collect_his_data.py
```

#### 2. 运行预测
```bash
python backtest_2025_ensemble_inverse.py
```

#### 3. 检查信号
查看最后一行输出：
```
Week XX (YYYY-MM-DD):
  Consensus: X/3
  Confidence: XX.X%
  Raw Prediction: 上涨/下跌
  Final (Inverse): 看多/看空
```

#### 4. 决策树
```
是否交易？
├─ 共识度 = 2/3 且 置信度 ≥ 70% → ✅ 高质量信号（历史100%）
├─ 共识度 = 3/3 且 置信度 ≥ 75% → ⚠️ 谨慎考虑（历史66.7%）
└─ 其他情况 → ❌ 不交易

如果是2/3共识：
  ✓ 执行交易
  ✓ 5-10%仓位
  ✓ 设置止损（-5%）

如果是3/3共识：
  阶段1-2：跳过（风险高）
  阶段3：如果置信度>78%可考虑，但仓位减半
```

#### 5. 执行交易（如果信号有效）
```
看多信号：
- 买入BTC现货
- 或做多合约（1-2x杠杆，不建议超过3x）

看空信号：
- 卖出BTC现货（如有持仓）
- 或做空合约（1-2x杠杆）
- 或保持USDT观望

记录：
- 日期
- 信号类型
- 入场价格
- 仓位大小
- 共识度/置信度
```

### 下周日（7天后）

#### 6. 结算上周交易
```
计算收益：
- 实际涨跌幅
- 预测是否正确
- 实际盈亏（扣除手续费）

更新记录表格：
| 日期 | 信号 | 入场价 | 出场价 | 涨跌% | 正确? | 盈亏% |
```

#### 7. 触发止损检查
```
如果单笔亏损 > 5%:
  ✓ 立即平仓
  ✓ 记录原因
  ✓ 等待下周新信号

如果本周出现极端波动（±15%）:
  ⚠️ 考虑提前平仓
  ⚠️ 系统设计针对正常市场
```

## 🛡️ 风险管理规则

### 强制止损条件（必须执行）
1. **单笔止损**：亏损达到-5%立即平仓
2. **连败止损**：连续3笔失败，暂停交易1个月
3. **回撤止损**：总资金从峰值回撤20%，停止交易
4. **黑天鹅止损**：单日暴跌>20%，无论盈亏立即平仓

### 仓位管理
```
阶段1（模拟）：0%真实资金
阶段2（验证）：总资金10-20%，单笔5%
阶段3（正常）：总资金30-50%，单笔10%

永远不要：
❌ 满仓（100%）
❌ 高杠杆（>3x）
❌ 加仓摊平（亏损时加码）
```

### 心理纪律
```
✓ 信号出现就执行，不要犹豫
✓ 信号消失就平仓，不要贪婪
✓ 严格遵守止损，不要侥幸
✓ 记录所有交易，不要选择性记忆

❌ 不要因为一次盈利就加大仓位
❌ 不要因为一次亏损就怀疑系统
❌ 不要手动调整系统参数追求完美
❌ 不要和别人分享实时操作（避免心理压力）
```
